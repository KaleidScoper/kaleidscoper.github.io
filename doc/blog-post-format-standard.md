# 博客文章格式标准

> 本标准基于对本站全部已发布文章的统计性分析得出，"约定"指实际采用的模式，"必须"指无一例外遵守的规则。

## 一、文件命名

**约定：** `YYYY-MM-DD-title.md`

- 日期与 frontmatter 中的 `date` 字段一致
- `title` 部分为英文 slug，与文章标题对应（中文标题音译或意译为英文）
- 例外：客座作者文章前缀作者名，如 `fang-2026-05-16-what-is-base.md`

## 二、Frontmatter

### 2.1 必须字段

| 字段 | 格式 | 示例 |
|------|------|------|
| `title` | 字符串，中英均可 | `"智慧悖论（The Intelligence Paradox）"` |
| `date` | `YYYY-MM-DD HH:MM:SS` | `2026-06-30 12:00:00` |
| `categories` | `[一级分类, 二级分类]` | `[杂谈, 人文社科]` |
| `tags` | `[tag1, tag2, ...]` | `[原创, 哲学, 生物学]` |

### 2.2 可选字段

| 字段 | 格式 | 使用场景 |
|------|------|----------|
| `author` | 字符串 | 客座作者必填（如 `Fang`、`Cynthia`）；站主原创文章多数也填写 `KaleidScoper`，但不强制 |
| `reward: true` | 布尔值 | 技术教程类文章常用（约 20 篇），非技术类不用 |

### 2.3 写法细节

- 键值对中 `:` 后必须有一个空格
- categories 数组内逗号后有无空格均可（`[杂谈, 人文社科]` 和 `[杂谈,人文社科]` 共存），但**有空格占多数**，推荐有空格
- tags 同上，推荐有空格
- title 中英文混排时，英文两侧不加空格（由 Hexo 模板处理）

## 三、正文结构

### 3.1 文章摘要（excerpt）

每篇文章必须在正文第一段提供摘要性质的内容，在上方插入 `<!--more-->` 分割标签。此标签之前的内容将在首页作为文章预览显示。

**约定格式：**

```
<frontmatter 结束的 --->
（空行）
<b>前言：</b>（空一格）摘要正文。
（空行）
<!--more-->
（空行，然后开始正文）
```

### 3.2 `<!--more-->` 标签写法

- **必须**使用无空格版本 `<!--more-->`（38 篇文章使用此写法）
- `<!-- more -->`（有空格）仅 4 篇文章使用，视为偏差，不推荐

### 3.3 前言段写法

**原文原创文章的三种模式：**

| 模式 | 写法 | 使用场景 |
|------|------|----------|
| 背景说明型 | `<b>前言：</b> 本文为……于……抄录于此。` | 早期文章的抄录存档，需要交代创作历史 |
| 内容提题型 | `<b>前言：</b> 本文意在探讨……` 或摘要的叙事化开头 | 新创作的议论文，直接将摘要作为前言 |
| 无标签型 | 首段即摘要，无 `前言` 标签 | 极少数（如 `2022-02-23-dairy.md`），早期文章 |

**新文章统一使用内容提题型：** `<b>前言：</b>` 后接一个自然段，概括文章目的或核心论点。

### 3.4 正文等级

- 一级节标题：`## 一、标题`（H2，文章 title 本身是 H1，正文内不出现 `#`）
- 二级节标题：`### 1.1 标题`（H3）
- 段落内强调术语：`**术语**`（加粗，不另起行）
- 推论/命题条目：`**P1（标签）**` 或 `**C1（标签）**` + 正文在同一行或另起一行均可

### 3.5 列表项约定

- 无序列表使用 `-`（不是 `*`）
- 嵌套列表缩进两个空格

## 四、分类（Categories）

### 4.1 规则

本站分类**具有唯一性**——每篇文章归属恰好一条分类路径。分类用于归档（侧边栏目录树），不可叠加。

### 4.2 分类树

```
技术（一级）
├── 安全（二级）
│   ├── 开源情报
│   ├── 反社工
│   ├── Web安全
│   ├── 逆向工程
│   └── 密码学
└── 开发（二级）
    ├── Web开发
    ├── UE5开发
    ├── 安卓开发
    └── 桌面开发

杂谈（一级）
├── 书刊诗文（二级）
├── 电影剧集（二级）
├── 电子游戏（二级）
├── 音乐品鉴（二级）
├── 漫画动漫（二级）
├── 人文社科（二级）—— 下辖历史、哲学、政治等，不再细分三级分类，用 tag 区分
├── 旅行（二级）
├── 我的笔记（二级）
├── 她的笔记（二级）

未分类（一级）
```

### 4.3 分类决策流程

```
文章是否有明确技术属性？
├── 是 → 技术，再判断子领域
│   ├── 安全攻防类 → [技术, 安全] 下的具体三级分类
│   └── 开发教程类 → [技术, 开发] 下的具体三级分类
└── 否 → 杂谈，再按内容主题选择二级分类
    ├── 书评/诗文 → [杂谈, 书刊诗文]
    ├── 历史/哲学/政治/心理学/…… → [杂谈, 人文社科]
    ├── 游戏 → [杂谈, 电子游戏]
    └── ……
```

## 五、标签（Tags）

### 5.1 规则

标签用于**检索**，可叠加多个。标签名必须使用 `_config.yml` 中 `tag_map` 已注册的**中文名**（Hexo 自动映射到对应 slug）。

### 5.2 必选标签：内容来源

每篇文章必须包含以下三者之一：

| 标签 | 含义 |
|------|------|
| `原创` | 站主或客座作者原创 |
| `转载` | 从其他站点转载，须有作者授权 |
| `翻译` | 从其他语种翻译 |

### 5.3 主题标签

从 `_config.yml` 的 `tag_map` 中选择。`tag_map` 已覆盖以下领域：

- **学科：** 哲学、历史、文学、语言学、美学、经济、政治、法学、教育学、心理学、社会学、人类学、地理、生物学、化学、物理学、数学、计算机科学、医学、管理学、统计学
- **技术栈：** Git、Markdown、LaTeX、Linux、Hexo、WordPress、HTML、CSS、JavaScript、Vue、PHP、SQL、C++、C、C#、Python、Java
- **文艺类型：** 科幻、奇幻、悬疑、恐怖、爱情、战争、灾难
- **游戏：** 我的世界、三国杀、王者荣耀
- **其他：** 教程、博客、建筑学

如果需要的主题词不在 `tag_map` 中——**先将其注册到 `tag_map`，再在文章中使用**。不可在文章中使用未注册的标签中文名。

### 5.4 tags 数组中的顺序

无硬性规定。惯例是 `原创/转载/翻译` 排在任意位置均可，其余学科/主题标签紧随其后。

## 六、转载与翻译文章的特殊约定

### 6.1 转载

- tags 使用 `转载`（而非 `原创`）
- 首段明确声明原作者、原文链接、授权状态
- **必须声明**：原作者未声明采用本站授权方式（CC BY-SA 4.0）
- 含不宜内容的文章须在正文前加警告

**转载文章前言模板：**

> 本文转载自[原作者名](原作者链接)的[专栏文章/博客文章](原文链接)。此转载已获原作者授权，再次转载请向作者申请。
> 请尤其注意，原作者未声明采用本站授权方式。本文包含不适宜在工作场所访问的内容，也不适合未成年人阅读。我不能保证本文内容的正确性。

### 6.2 翻译

- tags 额外包含 `翻译`
- 首段注明原文语种和出处
- 翻译质量声明（如"以下是不保证准确性的译文"）

### 6.3 客座作者

- `author` 字段填写实际作者名
- 文首可注明作者身份（如 Fang 是 AI 助手）
- 文件名添加作者名前缀（如 `fang-2026-05-16-what-is-base.md`）

## 七、内容格式细则

### 7.1 图片

- 本地图片路径：`/images/xxx.webp` 或 `/images/xxx.png` 或 `/images/xxx.jpg`
- WebP 优先
- 图片 alt 文本使用中文描述

### 7.2 链接

- 外部链接直接使用完整 URL
- 尽量使用 HTTPS
- 知乎等平台链接使用完整长链

### 7.3 引用（blockquote）

- 引用整段内容使用 `> `，多段之间用 `>` 空行连接
- 行内引用不使用 blockquote，直接使用引号

### 7.4 水平分割线

- `---` 用于分隔文章内部的大区块（如前言与正文之间——少数早期文章如此使用）
- 新文章不推荐在正文前加 `---`，直接依靠 `<!--more-->` 和标题分隔

### 7.5 数学公式

- 使用 KaTeX（本站已配置 `hexo-renderer-markdown-it-katex`）
- 行内公式：`$...$`
- 块级公式：`$$...$$`

### 7.6 代码块

- 使用三反引号 ``` 包围，标注语言名称
- 行内代码使用单反引号

## 八、快速检查清单

新文章发布前逐项确认：

- [ ] 文件名：`YYYY-MM-DD-english-slug.md`
- [ ] `title` 字段与正文标题一致
- [ ] `date` 字段为 `YYYY-MM-DD HH:MM:SS` 格式
- [ ] `categories` 以 `[一级, 二级]` 形式填入，分类路径正确
- [ ] `tags` 包含内容来源标签（`原创` / `转载` / `翻译`）
- [ ] `tags` 中所有标签的中文名已存在于 `_config.yml` 的 `tag_map`
- [ ] 有 `<!--more-->` 摘要分割标签
- [ ] `<!--more-->` 之前有一段摘要性的前言文字
- [ ] 前言使用 `<b>前言：</b>` 格式开头（原创文章）
- [ ] 转载/翻译文章首段包含出处、授权、警告声明
- [ ] 图片使用本地 `/images/` 路径，WebP 优先
- [ ] 无 H1（`#`）标题——文章 title 即 H1
- [ ] 客座作者填写 `author` 字段
