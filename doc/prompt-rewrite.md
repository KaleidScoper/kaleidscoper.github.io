# Categories & Tags 页面样式优化 Prompt

## 背景与约束

本站使用 Hexo + Ayer 主题（已 fork 并自行维护，可直接修改主题源码，无需考虑上游兼容）。

全站预期视觉风格以 `/archives/` 页面为准：用排版空间（留白、缩进、字号差异）和简单线条（如时间线）传达信息层级，不使用卡片、彩色色块或装饰性图标/emoji，追求"文人书斋"式的克制与可读性。后文提到的所有辅助线条或图标，颜色均跟随当前亮色/暗色模式下的正文文字颜色。应考虑到架构上的优雅。

## Categories 页面（`/categories/`）

当前为毛玻璃卡片 + 无序列表。请改为以下布局：

### 每个一级分类渲染为一个"双栏块"

```
一级分类名称    │  二级分类A（5）
10 篇文章       │    三级分类甲（2）
                │    三级分类乙（3）
                │  二级分类B（8）
                │    三级分类丙（4）
                │    三级分类丁（4）
```

- **左栏**：一级分类名称，字号明显大于正文、字重加粗，下方紧跟一行小字"xx 篇文章"（取代原有的计数徽章）。类似 Archives 页面中"年份"节点的处理方式。
- **中间**：一条纵向实线，从该块顶部延伸到底部，作为左右栏的视觉分隔。
- **右栏**：二级、三级及更深层分类，保持层级缩进的无序列表形式。每个分类名称后紧跟全角括号内的文章计数，如"二级分类A（5）"。去除所有卡片、徽章样式。可带有能提升可读性的极简样式，如类似IDE的缩进辅助线或者tree命令风格的层级辅助线。
- 无需折叠/展开功能。

### 一级分类之间的间距

不同一级分类的双栏块之间应有充分的垂直间距（各块的纵向分隔线不能首尾相连），营造与 Archives 页面一致的"呼吸感"。整个区块不应呈现为可见的卡片——无背景色、无边框、无阴影、无圆角。

## Tags 页面（`/tags/`）

当前为渐变色毛玻璃卡片构成的标签云。请改为以下布局：

### 功能保留

保留现有的搜索框、标签总数统计、排序切换等功能。

### 单个标签块的样式

```
│ 标签名称
│ xx 篇文章
```

- 左侧一条短竖线（类似 blockquote 左边框的装饰线），右侧上方为较大字号的标签名称，下方为较小、较淡的"xx 篇文章"纯文本。
- 去除所有卡片效果（无背景色、无边框、无阴影、无渐变、无圆角）。

### 整体布局

- 所有标签块**等宽**，以 CSS Grid 网格排列，响应式适配不同屏幕宽度（高度允许随内容自适应）。
- 标签块之间的垂直与水平间距应与 Archives 等页面的留白节奏保持一致。
