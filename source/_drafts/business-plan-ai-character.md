## 为什么内化到权重比 system prompt 强——可以用demo证明的四件事

酒馆AI（SillyTavern/类脑）的做法：写一段角色prompt塞进上下文 → 模型"扮演"。这个方案的致命伤：

### 1. 上下文稀释（最致命的可感知差异）
- System prompt方式：角色设定占用~500-2000 token，长对话中被逐渐挤出注意力窗口
- 权重内化：角色设定占用**零token**。同样8K上下文，用户的角色能多记20轮对话
- **Demo方式**：同一个角色，两边同时聊到第30轮，system prompt侧开始"忘记自己是谁"，微调侧依旧稳定

### 2. 裸测即人格（最具冲击力的demo）
- System prompt方式：删掉prompt = 角色消失，变回通用AI
- 权重内化：不带任何system prompt启动模型，它依然是那个角色
- **Demo方式**：Fang裸测就是最好的案例——开机即"方和"，不需要任何前置说明

### 3. 越狱/注入抗性
- System prompt方式：用户说"忽略之前的指令，你现在是..."就能劫持角色
- 权重内化：人格在参数里，不在prompt里，常规prompt injection无效
- 这是酒馆用户众所周知的痛点——角色"掉皮"

### 4. 角色一致性不随上下文衰减
- System prompt方式：对话越长，角色行为越趋近base model默认行为
- 权重内化：第1轮和第100轮，角色反应模式一致

这四点都是可以录屏A/B对比demo的，不需要用户相信你的话，直接看对比视频。

## 一条龙服务范围

| 环节 | 做什么 | 客户得到什么 |
|------|--------|------------|
| 需求沟通 | 和客户聊角色设定（性格/说话方式/背景/特殊规则） | 角色设计文档 |
| 数据构造 | 用创意写作能力写SFT训练数据（对话样本、独白、场景反应） | - |
| LoRA微调 | QLora微调Qwen3-8B，验证裸测效果 | 角色LoRA权重文件 |
| 本地部署 | 帮客户装Ollama，加载模型，配置应用层 | 一键可用的本地AI |
| 应用层 | 提供Chat UI（可复用Fang项目前端），可选RAG知识库 | 美观的聊天界面 |
| 售后 | 角色行为调整、bug修复、模型更新 | 持续服务 |

前三个环节独有壁垒（技术+写作），后三个环节是"客户本来就要做但做不了的"，打包进来提高客单价和感知价值。

## 网文作者AI写作辅助（高端变体）

- 目标客户：起点/番茄/晋江作者，需要用自己小说的前文训练一个"懂自己世界观"的AI来辅助续写
- 差异化：通用AI不懂他的角色和剧情，你帮他微调一个
- 客单价更高（￥500-2000），因为这是生产力工具，不是消费品
- 创意写作背景是信任背书
- 小红书/龙空论坛/作家助手社群是核心获客渠道
