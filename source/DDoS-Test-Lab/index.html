实验参数速查
  • 实验平台: Raspberry Pi 4B (ARM Cortex-A72, 4GB) + Docker macvlan
  • 客户端: 20 合法容器 (192.168.1.128–.147) + 最多 50 攻击容器 (.150–.199)
  • 防御控制器: Python 3.11，1 Hz 决策循环
  • 内核执行: nftables 1.0.6，事务性增量下发
  • UDP Flood 工具: hping3 --flood --udp
  • SYN Flood 工具: hping3 --flood -S
  • 拦截率定义: drop_pps / total_pps（稳态 t>60s 均值）
  • 合法吞吐定义: legit_pass_pps / (20 × baseline_pps_per_client)
Ablation Study — Intercept Rate & Legitimate Throughput
消融实验说明
A 完整方案：CUSUM 全局检测 + IP 信誉动态维护 + nftables 四档分级限速  |  拦截率 99.5%，合法吞吐 97%
B 无 CUSUM：移除全局态检测，始终以普通态阈值运行  |  拦截率降至 97.5%，合法吞吐 90%
C 无信誉层：移除 IP 信誉系统，仅依赖固定限速规则  |  拦截率降至 93.0%，合法吞吐 94%
D 开题方案：仅单层限速，无 CUSUM、无信誉  |  拦截率 91.0%，合法吞吐仅 78%
NS-3 Scalability — Intercept Rate & Alert Latency
NS-3 扩展性实验说明
固定每攻击节点发包速率 1000 pps，横向扩展攻击节点数（50 → 500）。
评估 CUSUM 检测时延与拦截率随攻击规模的变化趋势。
结果：即使扩展至 500 节点（总攻击流量 500,000 pps），拦截率仍维持 99.75%，告警时延仅增至 1.8 s。
