# 技术美术（Technical Artist, TA）入门简要教程

---

## 一、TA 的定位与方向拆分

### 1. TA 的桥梁角色
- TA 是在 “美术 ↔ 程序/引擎” 之间的桥梁：协助美术更高效地做资源 / 管道 / 效果 / 工具。  
- TA 的工作通常包括：**工具开发 / 脚本 /资源导出 /渲染 /特效 /性能优化 /流程自动化** 等。  
- 在不同公司、不同项目里，TA 的强调点会偏向某一块（如以渲染 + 特效为主，或以管道工具开发为主）。

### 2. 常见细分方向

| 方向 | 主要技能 / 关注点 | 典型任务举例 |
|---|---|---|
| 渲染 / Shader / 材质 | 图形学基础、光照模型、着色器语言（HLSL / GLSL / ShaderLab / CG） | 给游戏做特殊材质、PBR 光照、后处理特效等 |
| 工具 / 脚本 / 管道 | 脚本语言（Python / C# / Lua）、自动化 | 写批量资源导出脚本、UI 界面工具、资源检查工具 |
| 特效 / VFX | 粒子系统、Shader + 粒子混合、后处理 | 火焰、爆炸、魔法特效等 |
| 优化 / 性能分析 | 渲染优化、内存优化、帧率分析 | 降低 draw call、合批、贴图压缩、内存使用优化 |
| 骨骼 / 绑定 / 动画管道 | 绑定系统、蒙皮、骨骼动画、绑定脚本工具 | 自动生成骨骼、绑定、动作混合、动画导出工具 |

## 二、入门阶段 — 技能基础建设

### 阶段 0：准备基础
- 美术基础：3D 模型、UV、贴图、法线贴图、材质、光照、色彩、构图  
- 图形学基础：向量／矩阵运算、光照模型、纹理采样、坐标变换  
- 编程基础：至少掌握一门脚本语言（Python / C# / Lua），熟悉算法、数据结构、面向对象编程  

### 阶段 1：涉猎各方向，摸索兴趣
- 在游戏引擎（Unity / Unreal）尝试：
  - 写简单 shader（漫反射 + 高光 + 法线贴图）  
  - 写编辑器脚本 / 工具  
  - 做小型粒子特效  
  - 用 Profiler 分析场景帧率瓶颈  

### 阶段 2：确定方向 & 系统深入

| 方向 | 深入建议 / 项目练手 |
|---|---|
| 渲染 / Shader | 系统学习 Shader 语言 + 渲染管线，做材质 / 后处理特效 |
| 工具 / 管道 | 写导出 / 检查工具，做资源管理器、批量处理脚本、编辑器 UI 插件 |
| 特效 / VFX | 做复杂粒子系统（多发射器、Shader 混合、贴图动画、曲线控制） |
| 性能优化 | 拿有规模的场景，使用 Profiler 找瓶颈，做优化（合批、贴图压缩、LOD、Occlusion） |
| 绑定 / 动画 | 写自动绑定脚本、蒙皮工具、骨骼导出 / 导入工具，做动画混合 / IK / FK /蒙皮优化 |

### 阶段 3：进阶 + 求职准备
- 做一个完整 demo 项目（效果 + 工具 + 优化）  
- 写博客 / 总结 / 开源部分工具，积累 Portfolio  
- 针对公司招聘要求补技能（Unity / Unreal / Maya / Houdini / shader / Python）  
- 尝试实习 / Game Jam 担任 TA / 技术美术角色

## 四、简化三月入门行动计划

| 月份 | 每周目标 / 核心任务 |
|---|---|
| 第 1 个月 | 补图形学基础 + 美术基础，每天 1h；熟悉引擎基本操作、材质系统、Shader；做一个简单 shader |
| 第 2 个月 | 工具 / 脚本练习：小插件（导出 / 批量处理 / 命名）；小特效制作；使用 Profiler 分析优化 |
| 第 3 个月 | 选定方向做完整 demo 项目（效果 + 工具 + 优化）；撰写项目说明 / 技术总结 /博客；准备简历、申请实习或参与 Game Jam |

## 五、备注
- 可以按阶段跳过自己已有基础  
- 核心目标：**先广度，再确定方向，最后做作品积累 Portfolio**  
- 实践与项目经验比单纯看教程重要
